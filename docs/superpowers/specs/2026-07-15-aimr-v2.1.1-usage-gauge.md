# AIMR v2.1.1: the gauge (design + ship record)

Date: 2026-07-15
Status: shipped in-session (founder ask: "a quick way to gauge how much
tokens used and left"). Extends the v2.1 doctor
(`2026-07-13-aimr-v2.1-doctor.md`); the no-ledger/no-spend-caps rule
stands untouched.

## Goal

v2.1 gave the skill eyes; v2.1.1 gives it a fuel gauge. The doctor could
already *fetch* quota (under `--deep`, bundled with liveness probes that
draw real quota), but the answer to "how much runway is left?" was buried
in an availability report, gated behind a flag that costs quota to run,
and expressed only as used-percent with a raw reset timestamp. The gauge
makes runway a first-class, one-command, zero-cost readout — for humans
(bars) and agents (`--json`) — without adding a second tool or violating
the two-layer rule.

One honesty constraint discovered up front and now load-bearing: **"tokens
used and left" is unanswerable in tokens.** Neither Claude nor Codex
exposes absolute token counts for subscription pools anywhere (OAuth
endpoint, statusline JSON, rollout snapshots: percent + reset time only —
verified against a live payload capture 2026-07-15). Tools that print
"tokens left" reconstruct from local transcript history and guess the cap
(ccusage `--token-limit max`). AIMR reports percent honestly instead.

## Research provenance

Two web-research agents, 2026-07-15 (~200k tokens; full reports in the
session transcript).

**Landscape survey** (ccusage, CodexBar, monclaude, caut, openusage,
tokscale, claude-powerline, ccstatusline, Claude Code built-ins):

- **Stateless pace exists in the field** — CodexBar renders "on pace /
  N% in deficit (runs out in ~T) / N% in reserve" from even-consumption
  budgeting against the current window (single reading; suppressed below
  3% window elapsed); ccstatusline overlays an elapsed-time cursor on its
  usage bars; monclaude ships a single-reading "unused weekly quota"
  nudge. Every *token-denominated* burn rate (ccusage tokens/min) needs
  transcript history. Even-burn pace is therefore the only burn signal
  compatible with AIMR's no-ledger rule — and the field's best tools
  already ship it.
- **Threshold consensus**: red at 90 is universal; 70 is the modal warn
  tier (Anthropic's own statusline docs and ccusage both use exactly
  70/90). Adopted verbatim.
- **Minimal loved readout** = per window: name, percent, time-to-reset.
  Agent-shaped surfaces speak used%; human-glance tools flip to left%.
  The gauge renders both.
- **Staleness is a field, not an error** in every credible tool (serve
  last-good + age marker) — matches the doctor's existing serve-stale
  design.

**Endpoint ground truth** (live OAuth payload captured on the reference
machine; codex protocol.rs archaeology across rust-v0.40.0 → main):

- OAuth usage endpoint: `utilization` confirmed 0–100 (the unified HTTP
  rate-limit *headers* use 0–1 — do not conflate); `resets_at` is ISO
  with fractional seconds (statusline JSON uses epoch seconds — same
  name, different type across surfaces); `limits[]` is the
  self-describing forward-compatible surface (kinds `session` /
  `weekly_all` / `weekly_scoped`) and carries a per-limit **`severity`**
  — the provider's own classification. Top-level per-model keys
  (`seven_day_opus`, experiment keys like `iguana_necktie`) are unstable;
  ignored in favor of `limits[]`.
- **The User-Agent header is load-bearing**: non-claude-code UAs land in
  an aggressively rate-limited bucket (claude-code#30930 ecosystem
  consensus). The stuck-429 bug is still open upstream (#30930 open;
  #31021/#31637 closed not-planned) — 180s cache + 30min failure backoff
  stays mandatory.
- **Window semantics are messier than v2.1 assumed**: the Claude 5h
  window is session-anchored but `resets_at` snaps to the top of the
  hour (inferred start ±1h); the 7d window is *contested* — 11 days of
  community polling showed utilization stepping down ~every 72h while
  `resets_at` sat a week out; codex windows are rolling ("start =
  reset − length" is not valid arithmetic; `resets_at` marks when the
  constraint clears). Consequence: pace is honest as a heuristic, and
  weekly pace is order-of-magnitude only — stated wherever pace appears.
- **Codex rollout shape has three eras**: legacy flat
  (`primary_used_percent`, ≤0.40), middle (`resets_in_seconds` relative
  to the event, 0.41–0.42), current (epoch `resets_at`, ≥0.50); plus
  `rate_limits` can be permanently null on some version/model combos
  (openai/codex#14880). The scanner now parses all three eras and
  tolerates null.

## Design

### One mechanism: a third speed, not a second tool

The consent ladder becomes three rungs with one new flag:

| Mode | Network | Quota drawn | Answers |
|---|---|---|---|
| default | none | none | what is routable (local knowledge) |
| `--usage` | free readouts only | **none** | how much runway per pool |
| `--deep` | everything | grok liveness draws the pool | both, live-verified |

`--usage` fetches the claude OAuth readout (politeness-cached) and the
codex rollout/app-server readout, skips gemini/grok liveness prompts
(those cost quota) *and* version subprocesses (node CLIs are slow; the
gauge doesn't render versions). `--json` trims to pools only — the
capability rollup is the full doctor's job, and `--usage`/`--deep` are
mutually exclusive flags. Exit codes: 0 = gauge produced, 1 = doctor
broke (routability is not assessed, so exit 2 does not apply).

### Derived fields: everything from a single reading

Every window everywhere (all three modes) now carries, beside the raw
`used_percent`/`resets_at`: `left_percent`, `resets_in_seconds`,
`already_reset` (reading predates its own window reset → percent is
stale-high, headroom suppressed), `pace` (= fraction-used ÷
fraction-of-window-elapsed, start inferred as reset − length; suppressed
below 5% elapsed — one notch more conservative than CodexBar's 3%),
`time_to_exhaustion_seconds` (linear extrapolation; the renderer warns
when it lands before the reset), and `headroom` (percent-pure: ≥70
tight, ≥90 critical). Claude windows additionally pass through the
endpoint's own `severity` verbatim — provider ground truth beside the
derived label. Pool-level `usage.headroom` names the worst window.

No history, no persistence beyond the existing politeness cache, no
gates. The report carries a `derived_fields_note` stating the heuristic
and its weak spots. Headroom is deliberately blind to reset proximity —
a critical window minutes from reset is self-healing — so
`resets_in_seconds` is always rendered beside it and SKILL.md says to
read them together. The doctor reports; the skill decides.

### Routing consumption (SKILL.md step 4)

Before any sizable delegation: run the gauge (cheap, cached), shift bulk
work off any pool whose `time_to_exhaustion_seconds <
resets_in_seconds`, trust server `severity` over derived labels, and
treat an under-used weekly window near its reset as use-it-or-lose-it
capacity (monclaude's nudge, generalized to routing). All soft signals;
substitutions stated, never silent.

## What shipped

1. `aimr_doctor.py`: `--usage` mode (probe policy split: net_usage /
   liveness / versions), window enrichment + `_pool_headroom`, gauge
   renderer (20-char bars, used·left·reset·pace per window, headroom
   summary line), `severity` passthrough, load-bearing User-Agent on the
   OAuth fetch (`AIMR_CLAUDE_UA` to override), middle-era codex
   `resets_in_seconds` recovery, `derived_fields_note` in every report.
   JSON changes are additive; `schema_version` stays 1.
2. `SKILL.md`: step 0 names the three speeds; step 4 gains the
   mid-session runway check and the headroom/pace/severity reading
   rules. Frontmatter description untouched (evals/triggers.json pins
   it).
3. `references/setup.md`: three-speed contract, "Reading the gauge"
   anatomy block, derived-field definitions with the window-semantics
   caveats, percent-is-the-honest-ceiling note, statusline-stdin
   alternative for in-session use.
4. `tests/test_doctor.py`: +9 hermetic tests — usage-mode subprocess
   canaries (no claude/gemini/grok spawns; codex limited to auth +
   app-server), derived-field math, pace suppression on young windows,
   already-reset flagging (stale-high readings can't drive headroom),
   human render, empty-machine exit 0, `--usage`/`--deep` exclusivity,
   middle-era codex shape, enrichment in local-mode codex windows.

## Explicitly deferred (and why)

- **Statusline extra**: now trivially wrappable (`--usage --json` +
  jq); monclaude owns the niche on the reference machine. Still an
  optional extra, still not core.
- **Time-of-day pace profiles** (CodexBar PR#401-style learned hourly
  bins): strictly better projections, but it is persisted history —
  banned from core by the no-ledger rule.
- **Unified rate-limit headers** (`max_tokens:1` probe → 0–1-fraction
  headers): draws quota and needs inference scope; the OAuth readout is
  free. Not worth it.
- **Gemini/grok standing readouts**: still no scriptable percent
  surface (unchanged from v2.1 research); gauge renders their honest
  one-liners.

## Review round (same session)

A 22-agent workflow review (high effort) surfaced 10 verified findings;
all fixed + regression-tested: (1) a stale rollout snapshot structurally
suppressed the free codex app-server readout in usage mode — now falls
through when no snapshot window is still live; (2) pace mixed timebases
(snapshot-age used% ÷ wall-clock-now elapsed), systematically understating
burn on aged readings — elapsed is now measured at the reading's `as_of`,
and expired exhaustion projections are omitted rather than reported
negative; (3) circular "needs --usage" advice when already running
--usage with token-less credentials; (4) contradictory "no live quota
readings" footer beneath rendered already-reset readings; (5) SKILL.md's
"politeness-cached" promise was only true for claude — the app-server
readout now shares the 180s cache; (6) gauge bars saturated at 97.5/2.5%;
(7) naive-ISO timestamps parsed as local time, now UTC; (8) usage-mode
JSON shape documented with `mode` as the discriminator; (9)+(10) the
app-server window parser unified into `_codex_windows` and the report
skeleton deduplicated. The fix round also live-caught the endpoint's
`severity` field carrying `"critical"` — the enum extends beyond the
`"normal"` research had observed.

## Success criteria

On the reference machine (claude authed, codex/gemini absent, grok
unauthenticated): `--usage` returns in ~1s with live 5h/7d/scoped bars,
left%, reset countdowns, pace, a headroom summary, and honest one-liners
for the other pools — zero quota drawn, exit 0. A second run within 180s
serves the politeness cache (same `as_of`). `python3 -m pytest tests/`
green on 3.10 and 3.12; the full doctor's behavior and exit codes
unchanged.
