# AIMR v2.1: the doctor (design)

Date: 2026-07-13
Status: approved direction (founder decision in-session 2026-07-13: usage
visibility moves from deferred-roadmap to shipped-in-core — as a *probe*,
not budget machinery; the no-ledger/no-spend-caps rule stands). Supersedes
the v2.1 sketch in `2026-07-11-aimr-v2-one-skill-pack-design.md` §Roadmap
and corrects that spec's stale research claims (see Corrections).

## Goal

Give the skill eyes. AIMR v2.0 *describes* pool health but cannot see it:
SKILL.md step 4 mandates a pool-health check that setup.md admits is manual,
and on the reference machine 3 of 4 lane CLIs aren't even installed — the #1
provider in 6 of 7 capabilities is silently dead. v2.1 ships one stdlib
script, `scripts/aimr_doctor.py`, that answers in under 2 seconds: which
CLIs are installed, which are authenticated, how much quota each pool has
left, and therefore which registry lanes are routable *right now*.

## Research provenance

11-agent workflow, 2026-07-13 (2 repo audits, 8 web researchers, 1 critic;
~940k tokens, findings archived in the session transcript). Key corrections
to the 2026-07-11 spec it produced:

- **"No multi-CLI aggregator exists today" is refuted**: ccusage (17.1k★,
  multi-agent), CodexBar (18.1k★, 59 providers), caut, openusage, tokscale
  all exist and are maintained. AIMR does not build another aggregator; it
  ships thin routing-grade probes and interoperates.
- **Codex HAS headless quota readouts** (two): rollout session files persist
  `token_count` events carrying server-header rate-limit snapshots, and
  `codex app-server` exposes JSON-RPC `account/rateLimits/read`.
- **Gemini's individual tiers died server-side 2026-06-18** (free, AI Pro,
  AI Ultra — account-tier-based, not version-based; consumers pointed at the
  closed-source Antigravity CLI `agy`). Only API-key, Vertex, and org Code
  Assist seats still route.
- **Grok is no longer flat-cost**: since June 2026 SuperGrok meters one
  shared weekly pool across Chat/Imagine/Voice/Build; the CLI stamps usage
  fields on headless JSON output since 0.2.97.
- **Router prior art validates the architecture**: learned routers died of
  staleness (RouteLLM frozen on a 2024 pair; "routing collapse" degeneracy),
  while config-declared policy routing (Arch-Router) beats benchmark-score
  routing on human preference. Production routers (OpenRouter, LiteLLM) win
  on *availability engineering*. Lesson adopted wholesale: two layers with
  different freshness contracts — slow-moving quality rankings in the
  registry, live-probed availability from the doctor, never mixed.

## Design

### The two-layer rule

1. **Quality layer** (registry.json): ranked providers, scores with
   suite+date, quota weights. Slow-moving, versioned, honest.
2. **Availability layer** (aimr_doctor.py): installed / authenticated /
   quota-remaining, probed at routing time. NEVER persisted into the
   registry.

### The doctor: two speeds

**Default run — local, no network, <2s.** Per pool: binary on PATH +
version (subprocess, hard timeout), auth state from credential files, and
any quota snapshot that is free to read locally (codex rollout scan). Zero
tokens, zero network, zero quota drawn.

**`--deep` — adds network probes**, each with its own timeout and an
explicit cost note:

| Pool | Deep probe | Cost |
|---|---|---|
| claude-sub | GET `api.anthropic.com/api/oauth/usage` (Bearer from `~/.claude/.credentials.json` `.claudeAiOauth.accessToken`, header `anthropic-beta: oauth-2025-04-20`) | free; politeness-cached |
| codex-sub | `codex app-server` JSON-RPC `account/rateLimits/read` (only when no rollout snapshot exists) | free; local process spawn |
| gemini | `gemini -p 'reply ok' --output-format json` liveness (exit 41 = auth-dead) | one tiny request |
| grok | `grok --no-auto-update -p 'ping' --output-format json --max-turns 1` liveness | draws the shared weekly pool — noted in output |

### Per-pool probe recipes (verified 2026-07-13)

- **claude**: auth = `.claudeAiOauth.expiresAt` (epoch ms) > now in
  `~/.claude/.credentials.json` (macOS fallback: Keychain item "Claude
  Code-credentials"); plan from `subscriptionType`/`rateLimitTier` and
  `~/.claude.json` `.oauthAccount`. Usage (deep) = the OAuth endpoint's
  `five_hour`/`seven_day.utilization` + `resets_at` + `limits[]`
  (`weekly_scoped` per-model caps) + `extra_usage`. Known upstream stuck-429
  bug (claude-code#30930/#31021/#31637) → mandatory politeness: 180s
  success-TTL cache, 30min backoff keyed on the last FAILED fetch, never
  clobber last-good, atomic mkdir lock. Cache lives in a per-user dir under
  the system temp dir (`aimr-<uid>`, mode 0700 — a shared dir would serve
  one user's account numbers to another), never in the skill dir (plugins
  are replaced on update). These numbers are **account-wide**
  (server-metered across machines/surfaces).
- **codex**: auth = `codex login status` exit 0/1 (mode string on stderr;
  `codex doctor --json` `checks["auth.credentials"]` is the fallback); plan
  = base64url-decode the `tokens.id_token` JWT middle segment in
  `$CODEX_HOME/auth.json` → claim `"https://api.openai.com/auth"
  .chatgpt_plan_type` (undocumented-but-stable; try/except → "unknown
  plan"). Usage (local, free) = reverse-scan the newest
  `$CODEX_HOME/sessions/YYYY/MM/DD/rollout-*.jsonl` for the last
  `event_msg`/`token_count` line with non-null `rate_limits`:
  `primary`/`secondary` `{used_percent, window_minutes, resets_at
  (unix-s)}` + `credits` + `plan_type`. These mirror `x-codex-*` server
  headers — account-truth **as of the last codex turn on this machine**;
  always surface the snapshot timestamp.
- **gemini**: auth = `~/.gemini/settings.json` `.security.auth.selectedType`
  (`oauth-personal` | `gemini-api-key` | `vertex-ai`) corroborated by
  `oauth_creds.json` (refresh_token present) / `google_accounts.json`
  `.active` / `GEMINI_API_KEY` env. **Policy knowledge encoded**:
  `oauth-personal` without an org Code Assist seat (`GOOGLE_CLOUD_PROJECT`)
  is server-blocked since 2026-06-18 → verdict `blocked`, dated, with the
  `--deep` liveness probe as the re-verification path. Never invoke `/stats`
  headlessly (it leaks to the model as a prompt). Exit 41 =
  FatalAuthenticationError. Per-run usage exists in headless JSON `stats`
  but there is no cheap standing quota readout on the API-key path (250
  req/day Flash-only free tier, vendor-documented).
- **grok**: auth = `$GROK_HOME/auth.json` (default `~/.grok/`) entry with
  non-expired `expires_at` (OIDC-scope-URL-keyed; ~7-day tokens,
  CLI-refreshed). `grok models` exit code is useless for auth (exits 0
  signed-out); the JSON error object on a real probe is the truth. No
  stable scriptable pool-percent readout (the gRPC-web billing endpoint
  CodexBar uses is unofficial; the specialist's own live test of the ACP
  billing method returned -32601) → usage verdict = "weekly shared pool,
  check grok.com Settings → Usage", plus per-run `usage`/`total_cost_usd`
  fields on headless JSON since 0.2.97. Always pass `--no-auto-update` in
  scripted runs. Tier gate: media tools hard-fail on free/X Basic with "Do
  not retry this tool" — detect and mark the lane unavailable.

### Output contract

Human mode: one status line per pool (verdict, version, auth, usage bars
with age), a per-capability rollup (best routable provider or the reason
none is), fix hints with exact commands (flutter/npm-doctor style), and an
informational list of detected-but-unrouted agent CLIs (candidate lanes).

`--json` mode: stable schema (`schema_version: 1`) —
`pools.<id>.{installed, version, auth:{status, method, plan}, usage:{source,
network, as_of, age_seconds, confidence, windows:[{name, used_percent,
resets_at}]}, verdict, fix}` + `capabilities.<key>.{best_available,
routable[], blocked_reason}` + `unrouted_clis_detected[]`. Verdicts:
`ready | unauthenticated | auth-expired | blocked | absent | unknown`.
Usage numbers always carry `source`, `as_of`, and `confidence` — the
registry honesty rule applied to probe output. Missing data is `null`,
never a guess.

Exit codes: `0` = report produced and ≥1 capability routable; `2` = report
produced, nothing routable; `1` = doctor itself failed. Timeouts on every
subprocess call; the doctor never blocks routing (SKILL.md falls back to
manual checks if it can't run).

### Freshness contract (one rule per signal)

| Signal | Freshness | Persisted? |
|---|---|---|
| installed/version | probed every run | never |
| auth state | probed every run (local file reads) | never |
| claude usage | politeness cache: 180s success / 30min error backoff | temp dir only |
| codex usage | as fresh as the last codex turn; timestamp surfaced | read-only (codex's own files) |
| registry scores | suite+date; stale scores are advisory | registry (the quality layer) |

### Consent posture

The doctor reads credential files locally and never prints secrets (only
expiry/plan/tier fields). The default run makes **zero network calls**.
`--deep` is the explicit opt-in for network probes, and the grok liveness
probe's quota cost is stated in the output. What is read and what is called
is documented in `--help`, the script docstring, and references/setup.md.

## Explicitly deferred (and why)

- **New lanes** (Antigravity `agy`, factory droid, opencode): each violates
  the gotchas-earned-from-real-runs rule two days after a deliberate
  descope. The doctor's unrouted-CLI detection surfaces them as candidates;
  they enter per CONTRIBUTING.md when a real run earns gotchas. (droid
  first: graded autonomy + fail-fast exits + per-run worktrees made it the
  survey's top add.)
- **Statusline**: monclaude owns the Claude statusline niche; a
  cross-provider line can wrap `aimr_doctor.py --json` later without core
  changes.
- **Spend accumulation / per-delegation cost attribution**: budget
  machinery — banned from core. Benchmarks may use per-run stats fields
  when suites return (v2.2).
- **Unofficial endpoints** (chatgpt.com wham/usage, grok gRPC-web billing):
  drift-prone; the specialists who found them recommend against defaulting
  to them.

## Success criteria

On a machine with only `claude` installed (the reference machine), the
default doctor run reports claude `ready` (with plan), codex/gemini/grok
`absent` with install hints, every capability's true routability, and exits
2 if nothing routes — in under 2 seconds with zero network calls. With all
four CLIs installed and authenticated, `--deep` shows live 5h/7d claude
utilization, codex primary/secondary window percentages with snapshot age,
and dated tier verdicts for gemini/grok. `python3 -m pytest tests/` stays
green on 3.10 and 3.12.
